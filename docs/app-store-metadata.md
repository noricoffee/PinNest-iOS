## 説明文（4000文字以内）

> **fold前（冒頭3〜4行）が最重要**。ユーザーが「もっと見る」を押す前に伝わる部分。

```
「後で読もう」と思ったページ、気になった画像、大切なPDF——保存したのに見つからない、そんなストレスをなくします。

pinNest は URL・画像・動画・PDF・テキストを1つのアプリにまとめるコンテンツマネージャーです。すべてのデータはデバイス内にのみ保存されます。

【5種類のコンテンツに対応】
・URL：ページのサムネイル・タイトルを自動取得して見やすく表示
・画像・動画：フォトライブラリから選択してそのまま保存
・PDF：ファイルをインポートして一元管理
・テキスト：メモや引用文を直接入力して保存

【Share Extension でかんたん保存】
Safari や他のアプリの共有メニューから pinNest を選ぶだけで、すぐにピン留めできます。その場でタイトルやメモを追加することも可能です。

【タグ & キーワード検索】
保存したコンテンツにタグを付けて分類。キーワード検索とタグフィルターを組み合わせることで、大量のピンの中から目的のものをすばやく見つけられます。

【完全オフライン動作】
すべてのデータはデバイス内にのみ保存されます。クラウド同期も外部サーバーへの送信もなく、プライバシーを守りながら安心して使えます。

【ダークモード対応】
システム設定に合わせた自動切り替えに加え、アプリ内でライト／ダークを手動設定することもできます。

【アクセシビリティ対応】
Dynamic Type・VoiceOver に対応し、文字サイズの変更やスクリーンリーダーでも快適に操作できます。
```

## スクリーンショット戦略

### 基本方針

- **1枚目が最重要**: 検索結果一覧でサムネイル表示される。アプリの価値を0.5秒で伝える
- **アプリ画面を主役にする**: キャッチコピーは添え物。画面の美しさ・情報量で勝負する
- **6枚構成**を推奨（最大10枚だが多すぎると飽きられる）

### レイアウト構成

```
┌─────────────────┐
│                 │
│  キャッチコピー   │  ← 画面高さの約15%（1〜2行、小さめ）
│                 │
├─────────────────┤
│                 │
│                 │
│  アプリ画面      │  ← 画面高さの約80%（主役）
│  （大きく）      │
│                 │
│                 │
└─────────────────┘
```

- アプリ画面を全体の **80%** に配置。画面の実際のコンテンツを最大限見せる
- テキストは**上部のみ**に1行で収める（下に帯を置かない）
- フォントはシステムフォント Medium〜Semibold。**Bold は使わない**（主張が強くなりすぎる）
- 背景はアプリの配色そのまま（白 or ダーク系）。別途帯を作らない
- キャッチコピーは**8字以内**（短ければ短いほど視線がすぐ画面に移る）

### 推奨スクリーンショット順

| 順番 | 画面 | キャッチコピー（案） | 訴求ポイント |
|------|------|-------------------|------------|
| 1 | ホーム（グリッド、URL・画像・PDF混在、コンテンツ豊富に） | 何でも、ここに。 | 一元管理をひと目で訴求 |
| 2 | Share Extension（Safariから共有中） | 2タップで保存。 | 最大の差別化機能 |
| 3 | タグ検索・絞り込み結果 | すぐ見つかる。 | 整理・検索性の訴求 |
| 4 | ピン作成（URL→サムネイル自動取得済みの状態） | 貼るだけで完成。 | 手間のなさを訴求 |
| 5 | ピン詳細（画像またはPDF） | 5種類を一元管理。 | コンテンツの幅を訴求 |
| 6 | ダークモードのホーム画面 | ダークも、もちろん。 | 品質感・完成度 |
